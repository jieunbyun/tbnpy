# EMA mainshock–aftershock example: code structure

This folder builds the Bayesian network described in `EMA_aftershock.tex` for
the Eastern Massachusetts (EMA) highway network and exercises inference under
component-state conditioning. The structure mirrors the `ABCDE/` example but
scales up to 129 edges and `K_max` aftershock slots.

## Layout

```
examples/EMA_aftershock/
├── __init__.py
├── structure.md              <- this file
│
├── data/                     <- EMA network topology (provided)
│   ├── nodes.json            <- {node_id: {x, y, population}}
│   ├── edges.json            <- {edge_id: {from, to, length_km, ...}}
│   └── probs_eq.json         <- single-shock survival/failure probs (binary)
│
├── rsr_res/                  <- pre-computed RSR refs for system state
│   ├── refs_up_1.pt,  refs_up_2.pt
│   ├── refs_low_0.pt, refs_low_1.pt
│   └── metrics.json
│
├── results/                  <- generated by scripts (created on demand)
│
├── l.py  m.py  k.py  r.py  v.py  a.py  d.py  x.py  s.py   <- one node type
├── s01_define_model.py       <- assemble Variable + probability objects
├── s02_forward_inference.py  <- unconditional forward sampling; stats + histograms
├── s03_cal_s_x0.py           <- estimate P(S_t | X_{0,n} = 0) by surgery
├── plot_network.py           <- draw network and L_0 support rectangle
└── test_variables.py         <- unit tests, 3-4 cases per method
```

## BN node types and their files

Each file defines one or two probability classes that follow the standard
tbnpy custom-CPT contract (`__init__`, `sample(Cs_pars)`,
`log_prob(Cs)`). The "slot-aware" classes accept a `slot_idx` and use the
parent variable `K` to decide whether the slot is *active* (`slot_idx <= K`)
or *inactive* (delta at zero).

| File   | Class(es) | Children | Parents                          | Notes                                                                 |
| ------ | --------- | -------- | -------------------------------- | --------------------------------------------------------------------- |
| `l.py` | `L0`      | one of `L_x_0`/`L_y_0` | —                          | Uniform on `[low, high]` for one coordinate. Instantiate twice (one per axis) to cover the rectangle. |
|        | `Lt`      | one of `L_x_t`/`L_y_t` | `L_axis_0, R_t, V_t`       | Deterministic: `L_x_t = L_x_0 + R cos V` (`axis="x"`) or `L_y_t = L_y_0 + R sin V` (`axis="y"`). |
| `m.py` | `M0`      | `M_0`              | —                          | Truncated exponential on `[m_min, m_max]`.                            |
|        | `Mt`      | `M_t`              | `M_0, K`                   | Truncated exponential on `[m_min, M_0 - Δm]`, delta-at-0 if inactive. |
| `k.py` | `K`       | `K`                | `M_0`                      | Truncated Poisson; mean from the modified Omori law.                  |
| `r.py` | `R`       | `R_t`              | `K`                        | Radial decay on `[r_min, r_max]`, delta-at-0 if inactive.             |
| `v.py` | `V`       | `V_t`              | `K`                        | Uniform on `[0, 2π)`, delta-at-0 if inactive.                         |
| `a.py` | `A`       | `A_{t,n}`          | `M_t, L_x_t, L_y_t`        | Lognormal PGA (Campbell-1997 / Lee-2011). Edge midpoint is an init arg. |
| `d.py` | `D0`      | `D_{0,n}`          | `A_{0,n}`                  | Lognormal Park-Ang damage (Ghosh-2015 single shock).                  |
|        | `Dt`      | `D_{t,n}`          | `A_{t,n}, D_{t-1,n}`       | Truncated lognormal (lower bound = `D_{t-1,n}`, enforcing `D_t >= D_{t-1}`). Carries `D_{t-1,n}` if `A = 0`. |
| `x.py` | `X`       | `X_{i,n}` (binary) | `D_{i,n}`                  | Deterministic threshold indicator (`X = 1` iff `D < 0.4`).            |
| `s.py` | `S`       | `S_i` (states `0..max_st`) | `X_{i,1}, ..., X_{i,N}`    | RSR-based classifier (`rsr.classify_samples_with_indices`) with optional `s_fun` fallback for unknown samples. Unresolved samples receive `unknown_state` (default `-1`, deliberately outside the valid range). |

Locations are split into separate `L_x` and `L_y` `Variable`s **and**
separate probability objects, because the tbnpy inference engine
assumes one probability object per variable name. The split is
mathematically clean: `L_0`'s two coords are independent uniforms, and
`L_t`'s two coords are deterministic functions of the same parents
(`R_t, V_t`).

**System-state indexing.** `S_i` is 0-indexed: valid states are
`{0, 1, ..., max_st}`, matching the RSR convention where `refs_up_{s}`
(resp. `refs_low_{s-1}`) describes the boundary between `S >= s` and
`S <= s - 1`. The provided `rsr_res/` files give `max_st = 2`, so
`S_i in {0, 1, 2}`. Samples that the classifier cannot resolve (no
matching upper or lower ref at any level, and no `s_fun` supplied) are
labelled with the sentinel `-1` — chosen to sit outside the valid range
so unresolved cases can never be mistaken for state `0`.

## Scripts

### `s01_define_model.py` — model assembly

Provides the following helper functions:

* `load_topology(edge_subset=None)` — read `data/nodes.json` and
  `data/edges.json`, compute edge midpoints.
* `load_rsr_refs(max_st=2, device="cpu")` — load the upper/lower reference
  tensors that drive the multi-state classifier in `s.py`.
* `make_s_fun(dests=None)` — build and return an `s_fun` callable (uses
  `ndtools.fun_binary_graph.eval_population_accessibility`) that resolves
  component states left unknown by the RSR classifier. Loads the graph from
  `data/` once at construction time. Default destinations: `["n22", "n66"]`.
* `define_variables(edge_names, K_max)` and `define_probs(..., s_fun=None)` —
  build the `varis` and `probs` dicts. `s_fun` is forwarded to every `S` node.
  Variable keys follow:

    L_x_0, L_y_0, M_0, K, S_0
    A_0_{n}, D_0_{n}, X_0_{n}                       for n in edges
    R_{t}, V_{t}, L_x_{t}, L_y_{t}, M_{t}, S_{t}    for t in 1..K_max
    A_{t}_{n}, D_{t}_{n}, X_{t}_{n}                 for t in 1..K_max, n in edges

Running the file as a script does a smoke test (`K_max = 2`, full edge set,
just instantiates everything and prints counts).

**Example usage:**

```bash
cd examples/EMA_aftershock
python s01_define_model.py
```

### `s02_forward_inference.py` — unconditional forward sampling

Samples all BN variables with no evidence and saves:

* `results/forward_stats_Kmax{K}_maxst{M}_n{N}.csv` — long-form summary
  statistics. One row per state for discrete variables (`variable`, `type`,
  `state`, `prob`, `count`); one row per variable for continuous (`variable`,
  `type`, `mean`, `std`, `n`).
* `results/histograms/Kmax{K}_maxst{M}_n{N}/` — one PNG per variable,
  organised into sub-directories:

  | Sub-directory | Contents |
  | ------------- | -------- |
  | `global/`     | `L_x_0`, `L_y_0`, `M_0`, `K` |
  | `S/`          | `S_0` … `S_{K_max}` |
  | `A/`          | `A_{t}_{n}` for all `t`, `n` |
  | `D/`          | `D_{t}_{n}` for all `t`, `n` |
  | `X/`          | `X_{t}_{n}` for all `t`, `n` |
  | `t{i}/`       | `R_i`, `V_i`, `L_x_i`, `L_y_i`, `M_i` |

**Online accumulators.** Mean and variance are updated with Welford's
parallel algorithm so batching does not affect numerical accuracy.
Histogram bin edges are fixed from the first batch and reused, keeping
memory usage proportional to the number of variables, not the sample size.

**Caching.** If both the CSV and histogram directory already exist the
script exits early. Pass `force_recompute=True` to redo.

**Example usage:**

```bash
cd examples/EMA_aftershock
python s02_forward_inference.py
```

### `plot_network.py` — visualise the network and `L_0` support

Draws all 74 nodes (sized by population) and 129 edges, and overlays the
bounding rectangle used as `L_0`'s uniform support. The rectangle is the
axis-aligned bounding box of the node coordinates, expanded by
`margin_frac` (default `0.20`) on each side. The figure is saved to
`results/ema_network.png`.

**Example usage:**

```bash
cd examples/EMA_aftershock
python plot_network.py
```

To use a different margin, edit the call to `plot_network(margin_frac=...)`
in the `__main__` block, or import the function from another script.

### `s03_cal_s_x0.py` — conditional inference by CPT surgery

For each edge `n`, estimates `P(S_t | X_{0,n} = 0)` for all `t ∈ {0, ..., K_max}`.

Conditioning trick: `X_{0,n} = 0` is equivalent to `D_{0,n} >= 0.4`. Rather
than running rejection sampling or MCMC, the script overwrites the CPT of
`D_{0,n}` with a degenerate `DeltaD` distribution that places all mass at
`D = 1`, then does plain forward sampling. The substitute keeps the
original parents (so the topological sort still works); they are simply
ignored.

Outputs a long-form CSV
`results/S_given_X0_zero_Kmax{K}_maxst{M}_n{N}.csv` with columns
`edge`, `t`, `S`, `P(S_t=S | X_0_n=0)`, `cov`.

The `cov` column is the coefficient of variation of the empirical
proportion — closed-form binomial standard error divided by the
estimate: `sqrt((1 - p) / (n_sample * p))` (infinity when `p == 0`).
Older cached CSVs without the column are backfilled on load.

**Caching.** The CSV is written incrementally — one append per edge —
so an interrupted run keeps its partial work. If the file already
exists, `main()` loads it, skips edges already present, and only
processes the missing ones. Pass `force_recompute=True` to ignore the
cache and start fresh. The cache key is `(K_max, max_st, n_sample)`;
changing any of these produces a separate file.

**Batching.** `main()` and `estimate_S_marginals` accept `batch_size`
(default `None` = a single batch of `n_sample`). When set, the
`n_sample` draws are split into chunks of at most `batch_size`; raw
counts are accumulated across batches and only normalised at the end,
so the result is statistically identical to a single big batch — only
peak memory differs. Use this for large sample budgets (`n_sample` in
the hundreds of thousands or millions) that would otherwise OOM in the
BN's per-node `(1, n_sample, …)` allocations.

The `__main__` block runs a tiny demonstration over three target
edges so the script terminates in a few seconds. For the full 129-edge
sweep, drop `edge_subset` from the call.

**`edge_subset` semantics.** The BN itself is always built over the
**full** edge set — the RSR refs require this, since they encode the
system function over all 129 edges and the classifier `samples @ refs.T`
matmul would fail on a partial set. `edge_subset` only restricts which
edges the conditioning loop iterates over (i.e. for which `n` we
compute `P(S_t | X_{0,n} = 0)`).

**Example usage:**

```bash
cd examples/EMA_aftershock
python s03_cal_s_x0.py
```

To run on a GPU, prefix with `USE_CUDA=1` (PowerShell: `$env:USE_CUDA=1`).

### `test_variables.py` — unit tests

Hand-written tests (no `pytest` dependency). For each probability class and
each method (`sample`, `log_prob`) it provides 3-4 cases:

* 2 cases that show the result moving with an input (e.g. larger `M_0`
  shifting `K` upward, smaller region tightening `L_0`'s log density);
* 1-2 boundary cases (delta-at-0 slots, degenerate intervals, threshold
  edges, empty refs that route through `s_fun`).

**Example usage:**

```bash
cd examples/EMA_aftershock
python test_variables.py
```

Each test prints `PASS` / `FAIL` and a summary line; the script exits with
status `1` if any test fails.

```
                         └──────────────────────┘
```

## Dependencies

* `torch`, `numpy`, `pandas` — standard.
* `tbnpy` — the parent repo, found via `sys.path` insertion of the repo
  root.
* `rsr` — the sibling repo at `../../rsr/` is expected. `s.py` falls back
  to `classify_samples_with_indices = None` if the import fails, but then
  any instantiation of the `S` class will assert. The convenience scripts
  add `../../rsr` to `sys.path` automatically.
