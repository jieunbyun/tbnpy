# EMA mainshock–aftershock example: code structure

This folder builds the Bayesian network described in `EMA_aftershock.tex` for
the Eastern Massachusetts (EMA) highway network and exercises inference under
component-state conditioning. The structure mirrors the `ABCDE/` example but
scales up to 129 edges and `K_max` aftershock slots.

## Layout

```
examples/EMA_aftershock/
├── __init__.py
├── structure.md              <- this file
│
├── data/                     <- EMA network topology (provided)
│   ├── nodes.json            <- {node_id: {x, y, population}}
│   ├── edges.json            <- {edge_id: {from, to, length_km, ...}}
│   └── probs_eq.json         <- single-shock survival/failure probs (binary)
│
├── rsr_res/                  <- pre-computed RSR refs for system state
│   ├── refs_up_1.pt,  refs_up_2.pt
│   ├── refs_low_0.pt, refs_low_1.pt
│   └── metrics.json
│
├── results/                  <- generated by scripts (created on demand)
│
├── l.py  m.py  k.py  r.py  v.py  a.py  d.py  x.py  s.py   <- one node type
├── s1_define_model.py        <- assemble Variable + probability objects
├── s02_cal_s_x.py            <- estimate P(S_t | X_{0,n} = 0) by surgery
├── plot_network.py           <- draw network and L_0 support rectangle
└── test_variables.py         <- unit tests, 3-4 cases per method
```

## BN node types and their files

Each file defines one or two probability classes that follow the standard
tbnpy custom-CPT contract (`__init__`, `sample(Cs_pars)`,
`log_prob(Cs)`). The "slot-aware" classes accept a `slot_idx` and use the
parent variable `K` to decide whether the slot is *active* (`slot_idx <= K`)
or *inactive* (delta at zero).

| File   | Class(es) | Children | Parents                          | Notes                                                                 |
| ------ | --------- | -------- | -------------------------------- | --------------------------------------------------------------------- |
| `l.py` | `L0`      | `L_x_0, L_y_0`     | —                          | Uniform over a rectangle covering all nodes.                          |
|        | `Lt`      | `L_x_t, L_y_t`     | `L_x_0, L_y_0, R_t, V_t`   | Deterministic: `L_t = L_0 + R_t (cos V_t, sin V_t)`.                  |
| `m.py` | `M0`      | `M_0`              | —                          | Truncated exponential on `[m_min, m_max]`.                            |
|        | `Mt`      | `M_t`              | `M_0, K`                   | Truncated exponential on `[m_min, M_0 - Δm]`, delta-at-0 if inactive. |
| `k.py` | `K`       | `K`                | `M_0`                      | Truncated Poisson; mean from the modified Omori law.                  |
| `r.py` | `R`       | `R_t`              | `K`                        | Radial decay on `[r_min, r_max]`, delta-at-0 if inactive.             |
| `v.py` | `V`       | `V_t`              | `K`                        | Uniform on `[0, 2π)`, delta-at-0 if inactive.                         |
| `a.py` | `A`       | `A_{t,n}`          | `M_t, L_x_t, L_y_t`        | Lognormal PGA (Campbell-1997 / Lee-2011). Edge midpoint is an init arg. |
| `d.py` | `D0`      | `D_{0,n}`          | `A_{0,n}`                  | Lognormal Park-Ang damage (Ghosh-2015 single shock).                  |
|        | `Dt`      | `D_{t,n}`          | `A_{t,n}, D_{t-1,n}`       | Multi-shock lognormal. Carries `D_{t-1,n}` if `A = 0`.                |
| `x.py` | `X`       | `X_{i,n}` (binary) | `D_{i,n}`                  | Deterministic threshold indicator (`X = 1` iff `D < 0.4`).            |
| `s.py` | `S`       | `S_i` (3-state)    | `X_{i,1}, ..., X_{i,N}`    | RSR-based classifier (`rsr.classify_samples_with_indices`) with optional `s_fun` fallback for unknown samples. |

Locations are split into separate `L_x` and `L_y` `Variable`s because tbnpy
treats each `Variable` as scalar; a single CPT object simply lists both as
its children.

## Scripts

### `s1_define_model.py` — model assembly

Provides three helper functions:

* `load_topology(edge_subset=None)` — read `data/nodes.json` and
  `data/edges.json`, compute edge midpoints.
* `load_rsr_refs(max_st=2, device="cpu")` — load the upper/lower reference
  tensors that drive the multi-state classifier in `s.py`.
* `define_variables(edge_names, K_max)` and `define_probs(...)` — build the
  `varis` and `probs` dicts. Variable keys follow:

    L_x_0, L_y_0, M_0, K, S_0
    A_0_{n}, D_0_{n}, X_0_{n}                       for n in edges
    R_{t}, V_{t}, L_x_{t}, L_y_{t}, M_{t}, S_{t}    for t in 1..K_max
    A_{t}_{n}, D_{t}_{n}, X_{t}_{n}                 for t in 1..K_max, n in edges

Running the file as a script does a smoke test (`K_max = 2`, full edge set,
just instantiates everything and prints counts).

### `s02_cal_s_x.py` — conditional inference by CPT surgery

For each edge `n`, estimates `P(S_t | X_{0,n} = 0)` for all `t ∈ {0, ..., K_max}`.

Conditioning trick: `X_{0,n} = 0` is equivalent to `D_{0,n} >= 0.4`. Rather
than running rejection sampling or MCMC, the script overwrites the CPT of
`D_{0,n}` with a degenerate `DeltaD` distribution that places all mass at
`D = 1`, then does plain forward sampling. The substitute keeps the
original parents (so the topological sort still works); they are simply
ignored.

Outputs a long-form CSV `results/S_given_X0_zero_Kmax{K}_n{N}.csv` with
columns `edge`, `t`, `S`, `P(S_t=S | X_0_n=0)`.

The `__main__` block runs a tiny demonstration over three edges so the
script terminates in a few seconds. For the full 129-edge sweep, drop
`edge_subset` from the call.

### `test_variables.py` — unit tests

Hand-written tests (no `pytest` dependency). For each probability class and
each method (`sample`, `log_prob`) it provides 3-4 cases:

* 2 cases that show the result moving with an input (e.g. larger `M_0`
  shifting `K` upward, smaller region tightening `L_0`'s log density);
* 1-2 boundary cases (delta-at-0 slots, degenerate intervals, threshold
  edges, empty refs that route through `s_fun`).

Run with `python test_variables.py` from inside the folder.

## How the pieces fit together

```
       data/*.json           rsr_res/*.pt
            │                      │
            ▼                      ▼
   ┌────────────────────────────────────────┐
   │  s1_define_model.py                    │
   │     load_topology  load_rsr_refs       │
   │     define_variables  define_probs     │
   └────────────────────────────────────────┘
            │ varis, probs
            ▼
   ┌─────────────────────────┐    ┌─────────────────────────┐
   │  test_variables.py      │    │  s02_cal_s_x.py         │
   │  (per-class assertions) │    │  P(S_t | X_{0,n} = 0)   │
   └─────────────────────────┘    └─────────────────────────┘
```

## Dependencies

* `torch`, `numpy`, `pandas` — standard.
* `tbnpy` — the parent repo, found via `sys.path` insertion of the repo
  root.
* `rsr` — the sibling repo at `../../rsr/` is expected. `s.py` falls back
  to `classify_samples_with_indices = None` if the import fails, but then
  any instantiation of the `S` class will assert. The convenience scripts
  add `../../rsr` to `sys.path` automatically.
